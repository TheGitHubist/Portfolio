IDS Prototype for Linux

Project Overview

The IDS (Intrusion Detection System) Prototype is a Python-based solution designed for file integrity monitoring on Linux systems. It automates the process of tracking changes to critical files, ensuring system security and compliance by detecting unauthorized modifications.

How It Works

The system builds a JSON file that lists all files written by an administrator in a designated text folder. It then periodically checks these listed files for any changes. If no changes are detected, it generates a standard log entry. If modifications are found, it triggers an alert with the exact datetime of the change. A Linux service is implemented to automate the initial build of the JSON file (if it doesn't exist) and perform continuous checks, allowing administrators to simply start the service for hands-free operation.

Key Features

  • Automated JSON File Generation: Dynamically creates and maintains a list of monitored files
  • Real-time Change Detection: Monitors files for unauthorized modifications with precise timestamps
  • Comprehensive Logging: Generates detailed logs for normal operations and alerts for anomalies
  • Linux Service Integration: Seamless automation through systemd service for continuous monitoring
  • Administrator-Friendly: Simple service start for hands-off operation and maintenance

Context and Use Cases

Ideal for system administrators and security professionals managing Linux servers where file integrity is critical. Use cases include compliance monitoring, intrusion detection in production environments, and automated security auditing for sensitive directories.
