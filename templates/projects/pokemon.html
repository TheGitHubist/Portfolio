Pokemon on Linux Console

Project Overview

Pokemon on Linux Console is a multiplayer game project implemented in Python, featuring a client-server architecture. It recreates the classic Pokemon experience in a text-based console environment, allowing multiple players to connect and interact within a shared game world over a local area network (LAN).

How It Works

The system consists of three main Python files: client.py, server.py, and database.py. First, the database is initialized using database.py. Then, the server is launched. On separate machines within the same LAN, clients run client.py. Upon connection, users must login or register, with credentials verified or added by the server. Once authenticated, the server broadcasts menus to all connected clients and handles game options such as 'explore', 'go to pokecenter', 'go to the arena', and more, managing the shared game state.

Key Features

  • Multiplayer LAN Gaming: Connect multiple players on the same network for collaborative gameplay
  • User Authentication: Secure login and registration system handled server-side
  • Real-time Menu Broadcasting: Server synchronizes game menus and options across all clients
  • Console-Based Interface: Text-based gameplay reminiscent of classic console games
  • Modular Architecture: Separate client, server, and database components for easy maintenance

Context and Use Cases

This project demonstrates networking concepts, database management, and multiplayer game development in Python. It's ideal for learning about client-server architectures, LAN communication, and building interactive console applications. Suitable for educational purposes, game development workshops, or as a foundation for more complex multiplayer systems.
